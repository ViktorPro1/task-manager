# Task Manager — Frontend

Клієнтська частина побудована на **React + Vite**.  
Відповідає за інтерфейс користувача, авторизацію та взаємодію з backend API.

---

## Стек

- **React** — бібліотека для побудови UI
- **Vite** — інструмент збірки та dev сервер
- **React Router DOM** — клієнтський роутинг
- **Axios** — HTTP запити до backend API
- **Context API** — глобальний стан авторизації

## Опис файлів

### `main.jsx`

Точка входу React додатку. Рендерить кореневий компонент в DOM елемент `#root`. Обгортає додаток в три провайдери:

- `BrowserRouter` — для роутингу
- `AuthProvider` — для глобального стану авторизації
- `StrictMode` — для виявлення потенційних проблем під час розробки

---

### `App.jsx`

Головний компонент додатку. Визначає структуру роутів:

- `/login` → `LoginPage`
- `/register` → `RegisterPage`
- `/dashboard` → `DashboardPage` (захищений роут)
- `*` → редірект на `/login`

Містить компонент `PrivateRoute` — перевіряє наявність токена. Якщо токен відсутній — перенаправляє на `/login`.

---

### `src/api/axios.js`

Створює налаштований екземпляр Axios з базовим URL з `.env`. Додає **interceptor** на кожен запит — автоматично підставляє JWT токен з `localStorage` в заголовок `Authorization: Bearer <token>`. Завдяки цьому не потрібно вручну додавати токен в кожен запит.

---

### `src/context/AuthContext.jsx`

Глобальний контекст авторизації. Зберігає `token` і `user` в стані та `localStorage`. Надає два методи:

- **`login(token, user)`** — зберігає токен і юзера в `localStorage` і стані
- **`logout()`** — очищає `localStorage` і скидає стан

Хук **`useAuth()`** — зручний спосіб отримати доступ до контексту з будь-якого компонента.

---

### `src/pages/LoginPage.jsx`

Сторінка входу. Містить форму з полями `email` і `password`. При сабміті відправляє POST запит на `/api/auth/login`. У разі успіху — зберігає токен через `useAuth().login()` і перенаправляє на `/dashboard`. У разі помилки — показує повідомлення під формою. Містить посилання на сторінку реєстрації.

---

### `src/pages/RegisterPage.jsx`

Сторінка реєстрації. Містить форму з полями `name`, `email` і `password`. При сабміті відправляє POST запит на `/api/auth/register`. У разі успіху — зберігає токен і перенаправляє на `/dashboard`. У разі помилки — показує повідомлення. Містить посилання на сторінку входу.

---

### `src/pages/DashboardPage.jsx`

Головна сторінка додатку після авторизації. Відповідає за:

- **Відображення задач** — завантажує задачі з API при монтуванні і при зміні фільтра
- **Створення задачі** — форма з полями `title` і `description`
- **Зміна статусу** — кнопка перемикає задачу між `active` і `done`
- **Видалення задачі** — кнопка видаляє задачу з бази
- **Фільтри** — кнопки `Всі`, `Активні`, `Виконані` фільтрують список
- **Вихід** — кнопка викликає `logout()` і перенаправляє на `/login`

Використовує `useEffect` для завантаження задач при зміні фільтра і `useState` для локального стану форми та списку задач.

---

### `.env`

Файл з змінними середовища для Vite. **Не додається в Git.** Всі змінні повинні починатись з префіксу `VITE_` щоб бути доступними в коді.

```
VITE_API_URL=http://localhost:5000/api
```

---

### `package.json`

Конфігурація проекту. Містить залежності і скрипти:

- `npm run dev` — запуск dev сервера на `http://localhost:5173`
- `npm run build` — збірка для production
- `npm run preview` — перегляд production збірки локально

| `/register`  | RegisterPage         | Ні        |
| `/dashboard` | DashboardPage        | Так       |

## Взаємодія з API

Всі запити йдуть через `src/api/axios.js`. Токен підставляється автоматично через interceptor.

| Метод  | Endpoint         | Опис            |
| ------ | ---------------- | --------------- |
| POST   | `/auth/register` | Реєстрація      |
| POST   | `/auth/login`    | Логін           |
| GET    | `/tasks`         | Отримати задачі |
| POST   | `/tasks`         | Створити задачу |
| PUT    | `/tasks/:id`     | Оновити задачу  |
| DELETE | `/tasks/:id`     | Видалити задачу |
